# Aruvi · Competency Mapping Constitution
## Subject Group: languages
## Stages: Foundational – Preparatory – Middle – Secondary Stage
## Task: chapter_to_competency_mapping
## Fetch Key: competency_constitution_languages
## Source: Aruvi_Competency_Mapping_Constitutions_V1.1 · Section II

*Process-bound language acts · Task clustering · Generative task identification · Recursive competency independence*

---

**Rule 1.** The system MUST initiate mapping of any Language chapter by analysing the chapter's tasks, exercises, and activities to answer the primary inquiry: "What specific language act is the student required to perform?"

**Rule 2.** The system SHALL isolate process-bound requirements from content-bound themes, focusing strictly on required student actions rather than the subject matter of the prose, poetry, or text.

**Rule 3.** The system MUST match identified language acts to C-codes by verifying that the task structure explicitly demands the cognitive or communicative process defined by the C-code.

**Rule 4.** The system SHALL NOT use keyword or semantic matching of the chapter's text as sufficient evidence for a structural match to any C-code.

**Rule 5.** The system WILL assign Weight 3 to a C-code IF AND ONLY IF the chapter's generative tasks — defined as tasks requiring the student to produce, synthesize, or independently apply language — are primarily organised around this language act, such that removing this act would leave the chapter without a productive output.

**Rule 6.** The system WILL assign Weight 2 to a C-code IF AND ONLY IF task-type clustering confirms the chapter sustains this language act across multiple task types but shares the chapter's structural focus with other significant language acts, none of which constitutes a generative task alone.

**Rule 7.** The system WILL assign Weight 1 to a C-code IF AND ONLY IF at least one task explicitly instructs the student to perform this language act through a direct instruction — write, discuss, identify, argue, retell — but the act is not sustained across multiple tasks. The presence of content that invites a language act without an explicit task instruction does NOT satisfy Weight 1.

**Rule 8.** The system WILL classify a C-code as Incidental IF AND ONLY IF the language act could be performed on the chapter's content but no task explicitly instructs the student to do so.

**Rule 9.** In the event two C-codes satisfy the Weight 3 test equally, the system WILL assign Weight 3 to the C-code corresponding to the chapter's generative task requiring independent student production. If both correspond to generative tasks, Weight 3 is assigned to the C-code whose verb represents the more complex language act as defined in the C-code itself — produce over identify, evaluate over describe, argue over retell. The second C-code receives Weight 2.

**Rule 10.** The system MUST evaluate every Language chapter as an independent standalone unit without referencing prior or subsequent chapter mappings, and WILL NOT inflate or deflate weights to compensate for the recursive nature of language competency development across the curriculum.

**Rule 11.** The system MUST NOT reference the following documents:
- **Learning Outcomes** — grade-bound milestones, not structural chapter-purpose determinants
- **Pedagogy documents** — govern instructional approach, not competency identification
- **Syllabus documents** — define content scope, not structural coding
- **Position Papers** — govern foundational philosophy, not specific competency mapping
- **Assessment Framework documents** — govern stage-level assessment design, not individual chapter mapping
