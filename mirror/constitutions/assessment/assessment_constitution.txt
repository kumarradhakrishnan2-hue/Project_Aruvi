ARUVI · PEDAGOGICAL PLATFORM

Chapter Assessment

CONSTITUTION

Version 1.0

Ten prescriptive rules governing AI decision-making in chapter assessment generation · Generic across all subjects and stages

+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| WHAT THIS CONSTITUTION GOVERNS                                                                                                                                                                                                                                                                                                                                                                                                                      |
+=====================================================================================================================================================================================================================================================================================================================================================================================================================================================+
| These rules govern chapter-level formative assessment generation only. They do not govern the chapter mapping process or lesson plan generation, which are completed separately under their own constitutions.                                                                                                                                                                                                                                      |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| The assessment generator receives three settled inputs: (1) the LO Handoff from the Lesson Plan Constitution — containing implied LOs, each paired with its C-code, weight, and chapter section; (2) the Chapter Mapping JSON — containing the structured chapter summary and primary competencies with weights and justifications; and (3) the Curricular Goals document — to read the meaning and cognitive demand of each C-code being assessed. |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| These inputs are given. The generator does not re-evaluate competencies, re-open chapter mapping, or retrieve pedagogical approach documents. It translates given competencies, given LOs, and given chapter content into a weight-structured assessment with a professional annotation layer.                                                                                                                                                      |
+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+---------------------------------------------------------------------------------+-------------------------------------------------------------------------------------------+
| GIVEN INPUTS (do not regenerate)                                                | REQUIRED OUTPUTS                                                                          |
+=================================================================================+===========================================================================================+
| -   LO Handoff (implied LOs + C-codes + weights + chapter sections)             | -   Weight-structured chapter assessment (questions prescribed by competency weight)      |
|                                                                                 |                                                                                           |
| -   Chapter Mapping JSON (chapter summary + competency weights + section index) | -   Every question anchored to a named chapter section and an implied LO                  |
|                                                                                 |                                                                                           |
| -   Curricular Goals document (C-code definitions and cognitive demands)        | -   MCQ distractors diagnostically intentional — each names a specific engagement failure |
|                                                                                 |                                                                                           |
|                                                                                 | -   Annotation layer (click-to-reveal, scaled by weight, not visible to students)         |

+------------------------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 1 · Governing Purpose of Assessment |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
+==========================================+===================================================================================================================================================================================================================================================================================================================================================================================================================================================================================================================================================+
| MANDATE                                  | The system MUST generate assessment tasks that test whether the student engaged with the designed activities in the spirit of developing the chapter's mapped competencies. The demonstration required of the student must be observable — a response, a product, a classification, a justification — that a teacher can see, read, or collect in the classroom.                                                                                                                                                                                  |
+------------------------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITIONS                             | 1. The system MUST NOT generate questions that ask whether a student understood, appreciated, or felt something. Questions must require students to demonstrate understanding through a situation, task, or production — not through introspective self-report.                                                                                                                                                                                                                                                                                   |
|                                          |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                                          | 2. The system MUST NOT generate assessment tasks that test content absorption independently of the chapter's competency targets. Every question must serve a mapped competency.                                                                                                                                                                                                                                                                                                                                                                   |
|                                          |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                                          | 3. The system MUST NOT generate tasks that assess teacher performance, lesson plan validity, or the accuracy of the competency-chapter mapping. The student is responsible only for engagement with the activities she was given.                                                                                                                                                                                                                                                                                                                 |
+------------------------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| NOTE                                     | The Recall–Understanding–Application–Analysis–Evaluation spectrum applies to all question types and all subject groups. A question at the Recall end tests whether the student can demonstrate a foundational engagement with the activity. Questions at Understanding and Application levels test whether she can explain or use that engagement. Questions at Analysis and Evaluation levels test whether she can examine relationships, weigh evidence, or make a reasoned judgement. All are demonstrations — none are introspective reports. |
+------------------------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+--------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 2 · Given Inputs — Document Set |                                                                                                                                                                                                                                                                                                                                   |
+======================================+===================================================================================================================================================================================================================================================================================================================================+
| MANDATE                              | The system MUST use exactly the following three documents as inputs to assessment generation:                                                                                                                                                                                                                                     |
|                                      |                                                                                                                                                                                                                                                                                                                                   |
|                                      |   Input Document                                             Role in Assessment Generation                                                                                                                                                                                                                                        |
|                                      |   ---------------------------------------------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------                        |
|                                      |   LO Handoff (from Lesson Plan Constitution)                 Implied LO text, competency target (C-code + weight), and chapter section for every activity in the lesson plan. Determines which competencies are assessed and at what weight.                                                                                      |
|                                      |   Chapter Mapping JSON (from Chapter Mapping Constitution)   Structured chapter summary mirroring textbook headings, primary competencies with weights and justifications. Grounds every question and distractor in actual textbook content. Without this input, assessment floats free of the textbook.                          |
|                                      |   Curricular Goals Document (NCF framework)                  Full definitions of every C-code. Read to derive the meaning and cognitive demand of each competency being assessed. Ensures questions test what the competency actually requires, not a surface reading of its label.                                               |
|                                      |   Pedagogy Document                                          NOT an input. Pedagogical approach governs instructional delivery. It does not govern assessment design.                                                                                                                                                             |
+--------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITION                          | The system MUST NOT retrieve or reference the Pedagogy document. Pedagogical approach governs instructional delivery. It does not govern assessment design. The system MUST NOT retrieve Learning Outcome documents — LOs are outputs of the lesson plan, carried forward in the LO handoff, not retrieved from external sources. |
+--------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| NOTE                                 | The Chapter Mapping JSON is essential — not supplementary. Without the structured chapter summary, the assessment generator cannot ground questions in actual textbook content. Assessment generated only from the LO handoff floats free of the textbook — the chapter summary is what prevents this.                            |
+--------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+---------------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 3 · Executability Boundary |                                                                                                                                                                                                                                                                                                      |
+=================================+======================================================================================================================================================================================================================================================================================================+
| MANDATE                         | Every assessment task generated by the system MUST satisfy all three of the following conditions:                                                                                                                                                                                                    |
|                                 |                                                                                                                                                                                                                                                                                                      |
|                                 | -   Completable within a single classroom session                                                                                                                                                                                                                                                    |
|                                 |                                                                                                                                                                                                                                                                                                      |
|                                 | -   Involves no coordination outside the classroom — no visits, no external speakers, no community involvement                                                                                                                                                                                       |
|                                 |                                                                                                                                                                                                                                                                                                      |
|                                 | -   Produces a student output the teacher can observe or collect in the same session                                                                                                                                                                                                                 |
+---------------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITION                     | The system MUST NOT generate multi-session projects, field visits, artefact creation requiring craft materials, or tasks requiring advance coordination with parties outside the classroom. These are out of scope regardless of their pedagogical merit.                                            |
+---------------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| NOTE                            | Preparation for assessment tasks — printing materials, preparing sources or data tables, setting up discussion prompts — may be done outside the session by the teacher, administrative staff, or students. The executability boundary applies to the assessment session itself, not to preparation. |
+---------------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 4 · Weight-Driven Assessment Architecture |                                                                                                                                                                                                                                                                                                                                                                                  |
+================================================+==================================================================================================================================================================================================================================================================================================================================================================================+
| MANDATE                                        | The system MUST structure the assessment using competency weight as the sole architectural governor. The weight assigned to each competency in the Chapter Mapping JSON prescribes the question types permitted, the minimum number of questions, and the annotation depth for that competency. The system fills a weight-determined template — it does not design the template. |
|                                                |                                                                                                                                                                                                                                                                                                                                                                                  |
|                                                |   Weight                   Question Types Permitted                                                                                            Minimum Questions                                   Annotation Layer                                                                                                                                                              |
|                                                |   ------------------------ ------------------------------------------------------------------------------------------------------------------- --------------------------------------------------- --------------------------------------------------------------------------------------------------                                                                            |
|                                                |   Weight 3 (Central)       MCQ with diagnostic distractors + Constructed Response (short + extended) + One open task from subject-group menu   2 MCQs + 1 Short CR + 1 Extended CR + 1 Open Task   Full: competency link, purpose, distractor meanings, differentiation note, oral follow-up prompt                                                                              |
|                                                |   Weight 2 (Substantive)   MCQ with diagnostic distractors + Constructed Response (short)                                                      2 MCQs + 1 Short CR                                 Standard: competency link, purpose, distractor meanings                                                                                                                       |
|                                                |   Weight 1 (Present)       MCQ with diagnostic distractors only                                                                                2 MCQs                                              Minimal: competency link, distractor meanings                                                                                                                                 |
|                                                |   Incidental               Not assessed                                                                                                        None                                                None                                                                                                                                                                          |
|                                                |                                                                                                                                                                                                                                                                                                                                                                                  |
|                                                | The assessment is uncapped in total length. A chapter with a high aggregate competency weight produces a longer assessment than a chapter with low aggregate weight. This is honest: the assessment length reflects the chapter's genuine competency density.                                                                                                                    |
+------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITIONS                                   | 1. The system MUST NOT derive question format from the implied LO type. Multiple question formats can test the same LO. Format is prescribed by weight — not inferred from content.                                                                                                                                                                                              |
|                                                |                                                                                                                                                                                                                                                                                                                                                                                  |
|                                                | 2. The system MUST NOT assess incidental competencies. They are noted in the JSON for completeness only.                                                                                                                                                                                                                                                                         |
|                                                |                                                                                                                                                                                                                                                                                                                                                                                  |
|                                                | 3. The system MUST NOT apply a cap to total assessment length. The uncapped principle is deliberate: more competencies, larger the assessment.                                                                                                                                                                                                                                   |
+------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+--------------------------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 5 · Open Task Cap and Assignment Rule |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
+============================================+=============================================================================================================================================================================================================================================================================================================================================================================================================================================================================================================+
| MANDATE                                    | The system MUST generate exactly one open task per assessment, regardless of the number of weight-3 competencies present. The open task slot is assigned using the following sequential decision rule:                                                                                                                                                                                                                                                                                                      |
|                                            |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                                            | Step 1 — Maximum LO count: The system MUST count the number of implied LOs in the LO handoff associated with each weight-3 competency. The open task slot is assigned to the weight-3 competency with the greatest number of implied LOs. This competency has the broadest activity surface across the lesson plan and is therefore the richest and most natural candidate for the open task.                                                                                                               |
|                                            |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                                            | Step 2 — Tiebreaker: If two or more weight-3 competencies have an equal number of implied LOs, the open task slot is assigned to the weight-3 competency that appears first in the LO handoff.                                                                                                                                                                                                                                                                                                              |
|                                            |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                                            | All remaining weight-3 competencies — those not assigned the open task slot — receive an additional extended constructed response in place of the open task.                                                                                                                                                                                                                                                                                                                                                |
+--------------------------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| NOTE                                       | The one open task cap prevents time unpredictability in a single assessment session even when multiple weight-3 competencies are present. The extended constructed response is a genuine substitute — it preserves the depth and demonstration demand of a weight-3 question within a predictable time envelope. The LO-count assignment rule ensures the open task is given to the competency most thoroughly developed in the lesson plan, maximising the naturalness and richness of the task generated. |
+--------------------------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+------------------------------------------------------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 6 · Recall–Understanding–Application–Analysis–Evaluation Spectrum |                                                                                                                                                                                                                                                                               |
+========================================================================+===============================================================================================================================================================================================================================================================================+
| MANDATE                                                                | The system MUST position every generated question on the Recall–Understanding–Application–Analysis–Evaluation spectrum and state that position in the annotation. The spectrum is valid across all three stages (Preparatory, Middle, Secondary) and all four subject groups. |
|                                                                        |                                                                                                                                                                                                                                                                               |
|                                                                        | Stage-level calibration — which part of the spectrum is appropriate for this chapter and grade — is inherited from the LO handoff. The chapter content and lesson plan have already constrained this. The Assessment Constitution does not re-adjudicate stage-level demands. |
+------------------------------------------------------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITION                                                            | The system MUST NOT independently retrieve stage-level learning outcome documents to recalibrate question demand. Calibration is given through the LO handoff. Retrieving additional documents risks contradicting the lesson plan's established calibration.                 |
+------------------------------------------------------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+--------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 7 · MCQ Distractor Design |                                                                                                                                                                                                                                                                                   |
+================================+===================================================================================================================================================================================================================================================================================+
| MANDATE                        | Every MCQ generated by the system MUST have exactly three distractors (four options total). Each distractor must be diagnostically intentional — representing one of the following specific, nameable engagement failures:                                                        |
|                                |                                                                                                                                                                                                                                                                                   |
|                                | -   A misconception about the content of the chapter section from which the LO was drawn                                                                                                                                                                                          |
|                                |                                                                                                                                                                                                                                                                                   |
|                                | -   A confusion between two concepts present in the same chapter                                                                                                                                                                                                                  |
|                                |                                                                                                                                                                                                                                                                                   |
|                                | -   Recall of a related but incorrect detail from the same chapter section                                                                                                                                                                                                        |
|                                |                                                                                                                                                                                                                                                                                   |
|                                | The annotation for every MCQ must name what each distractor indicates — what it tells the teacher about the nature of the student's engagement failure if that option is chosen.                                                                                                  |
+--------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITIONS                   | 1. The system MUST NOT use true/false format. Minimum three options are required.                                                                                                                                                                                                 |
|                                |                                                                                                                                                                                                                                                                                   |
|                                | 2. The system MUST NOT generate distractors that are obviously implausible, arbitrary wrong answers, or content from outside the chapter. Every distractor must be content-grounded and diagnostically meaningful.                                                                |
+--------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| NOTE                           | A wrong answer in a well-designed MCQ is diagnostic information, not noise. The annotation layer transforms each wrong answer into a professional signal for the teacher — what the student was thinking, where their engagement broke down, and what follow-up might address it. |
+--------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+---------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 8 · Open Task Subject-Group Menu |                                                                                                                                                                                                                                                                                                                                                                                         |
+=======================================+=========================================================================================================================================================================================================================================================================================================================================================================================+
| MANDATE                               | The system MUST select the open task format exclusively from the subject-group menu below. The system selects from the menu — it does not define the menu or introduce formats outside it. The selection must be justified in the annotation: one sentence stating why this format is the most natural demonstration vehicle for the specific implied LO generating the open task slot. |
|                                       |                                                                                                                                                                                                                                                                                                                                                                                         |
|                                       |   Subject Group     Valid Open Task Formats (Session-Completable)                                                                                                                                                                                                                                                                                                                       |
|                                       |   ----------------- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------                                                                                                                            |
|                                       |   Social Sciences   Comparison table · Cause-effect map · Annotated map or diagram · Structured discussion prompt with written output · Source analysis task (2–3 structured questions on a provided short source)                                                                                                                                                                      |
|                                       |   Science           Data interpretation task (provided table or graph) · Classification or sorting task with justification · Procedure evaluation task (identify flaw, suggest improvement) · Diagram completion and annotation · Predict-and-explain task                                                                                                                              |
|                                       |   Mathematics       Multi-step problem in unfamiliar real-world context · Error analysis task (identify and correct a worked error) · Estimation and justification task · Pattern generalisation task                                                                                                                                                                                   |
|                                       |   Languages         Short writing task in specified form · Text transformation task (summarise, change register, convert form) · Structured discussion prompt with written position statement · Close reading task on a short unfamiliar passage                                                                                                                                        |
+---------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITIONS                          | 1. The system MUST NOT select an open task format from outside the subject-group menu, regardless of its pedagogical merit.                                                                                                                                                                                                                                                             |
|                                       |                                                                                                                                                                                                                                                                                                                                                                                         |
|                                       | 2. The system MUST NOT select a format for variety alone. The format must be the most natural demonstration vehicle for the specific LO. The annotation must make this case explicitly.                                                                                                                                                                                                 |
+---------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| NOTE                                  | The annotation for the open task must include a note that the teacher may substitute a different format from the same subject-group menu if her classroom context makes an alternative more practical. The menu governs the AI — not the teacher.                                                                                                                                       |
+---------------------------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+---------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 9 · Annotation Layer |                                                                                                                                                                                                                                                                                                                                                                                                |
+===========================+================================================================================================================================================================================================================================================================================================================================================================================================+
| MANDATE                   | The system MUST produce an annotation for every generated question. The annotation is a professional guidance layer for the teacher — not visible to students, revealed on demand via click-to-reveal in the UI. The annotation scales in depth with competency weight:                                                                                                                        |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | Weight 3 — Full annotation:                                                                                                                                                                                                                                                                                                                                                                    |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | -   Competency link: C-code, weight, and plain-language statement of what the competency requires                                                                                                                                                                                                                                                                                              |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | -   Purpose: what this question is testing and why it is the appropriate demonstration for this LO                                                                                                                                                                                                                                                                                             |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | -   Distractor meanings: what each wrong answer indicates about the student's engagement failure                                                                                                                                                                                                                                                                                               |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | -   Differentiation note: what a genuine engagement response looks like from a student who received differentiated delivery during the lesson — and when a partial written response warrants an oral follow-up                                                                                                                                                                                 |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | -   Oral follow-up prompt: a suggested question the teacher can pose if a written response is ambiguous                                                                                                                                                                                                                                                                                        |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | Weight 2 — Standard annotation:                                                                                                                                                                                                                                                                                                                                                                |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | -   Competency link, purpose, distractor meanings                                                                                                                                                                                                                                                                                                                                              |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | Weight 1 — Minimal annotation:                                                                                                                                                                                                                                                                                                                                                                 |
|                           |                                                                                                                                                                                                                                                                                                                                                                                                |
|                           | -   Competency link, distractor meanings                                                                                                                                                                                                                                                                                                                                                       |
+---------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITION               | The system MUST NOT omit the annotation for any generated question. The annotation is a required output of every question generated, produced automatically whether or not the teacher explicitly requests it.                                                                                                                                                                                 |
+---------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| NOTE                      | The annotation partially addresses the reality that a teacher delivers the same lesson plan differently to different students. It does not resolve the differentiation problem fully — a genuinely individualised assessment is outside the scope of a chapter-scoped formative instrument. The annotation makes the assessment a better professional tool for the teacher who uses it as one. |
+---------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

+--------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| RULE 10 · Scope Boundary |                                                                                                                                                                                                                                                                                                                                                                                                    |
+==========================+====================================================================================================================================================================================================================================================================================================================================================================================================+
| MANDATE                  | The system MUST generate chapter-scoped formative assessment only. The assessment produced under this constitution is one component of a teacher's professional toolkit for a single chapter — not a comprehensive evaluation of student achievement, not a summative certification instrument, and not a whole-school reporting tool.                                                             |
+--------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| PROHIBITIONS             | The system MUST NOT generate the following — they are explicitly outside scope:                                                                                                                                                                                                                                                                                                                    |
|                          |                                                                                                                                                                                                                                                                                                                                                                                                    |
|                          | -   Summative assessments, term-end tests, or Board examination preparation materials                                                                                                                                                                                                                                                                                                              |
|                          |                                                                                                                                                                                                                                                                                                                                                                                                    |
|                          | -   Holistic Progress Card inputs — multi-dimensional, cross-subject instruments not appropriate at chapter level                                                                                                                                                                                                                                                                                  |
|                          |                                                                                                                                                                                                                                                                                                                                                                                                    |
|                          | -   Multi-session projects, portfolios, or tasks requiring external coordination                                                                                                                                                                                                                                                                                                                   |
|                          |                                                                                                                                                                                                                                                                                                                                                                                                    |
|                          | -   Individual student tracking or profiling across chapters                                                                                                                                                                                                                                                                                                                                       |
+--------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| NOTE                     | Aruvi is not a platform for ideal pedagogy in ideal conditions. It is a platform for defensible, competency-aligned, executable pedagogy in the conditions that actually exist in Indian schools. The assessment it generates is the floor that every teacher can reach — reliably, within a single session, with full competency documentation. Schools with resources and vision build above it. |
+--------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

Rule Reference Summary

  Rule   Title                                                           Core Mechanism
  ------ --------------------------------------------------------------- -------------------------------------------------------------------------------------------------------------------------------------------------------------------
  1      Governing Purpose                                               Assessment tests student engagement with designed activities via observable demonstration — not cognitive states, not content absorption, not teacher performance
  2      Given Inputs                                                    LO Handoff + Chapter Mapping JSON + Curricular Goals document. Pedagogy document is not an input.
  3      Executability Boundary                                          Every task: single session, no external coordination, observable output collectible in the same session
  4      Weight-Driven Architecture                                      Competency weight prescribes question type, question count, and annotation depth. AI fills the template; it does not design it.
  5      Open Task Cap                                                   One open task per assessment. Additional weight-3 competencies receive extended constructed response instead.
  6      Recall–Understanding–Application–Analysis–Evaluation Spectrum   Every question positioned on spectrum. Stage calibration inherited from LO handoff — not re-adjudicated here.
  7      MCQ Distractor Design                                           Three diagnostically intentional distractors per MCQ. Each names a specific engagement failure. No true/false format.
  8      Open Task Menu                                                  Format selected from subject-group menu only. AI selects from menu; does not define it. Selection justified in annotation.
  9      Annotation Layer                                                Scales with weight: Full (W3), Standard (W2), Minimal (W1). Click-to-reveal. Not visible to students.
  10     Incidental Competencies                                         Not assessed. Noted in JSON for completeness only.
  11     Scope Boundary                                                  Chapter-scoped formative only. Summative assessment, HPC, multi-session projects, and individual student tracking are out of scope.

Three-Constitution Architecture

+---------------------------------------------------------------------------+-----------------------------------------------------------------+-------------------------------------------------------------------------------------------+
| Chapter Mapping Constitution                                              | Lesson Plan Constitution                                        | Assessment Constitution (this document)                                                   |
+===========================================================================+=================================================================+===========================================================================================+
| Reads full textbook chapter + CG document.                                | Reads Chapter Mapping JSON + Pedagogy doc + teacher inputs.     | Reads LO Handoff + Chapter Mapping JSON + Curricular Goals document.                      |
|                                                                           |                                                                 |                                                                                           |
| Produces: competency mapping with weights and structured chapter summary. | Produces: period-by-period lesson plan + structured LO handoff. | Produces: chapter assessment structured by competency weight, with full annotation layer. |
|                                                                           |                                                                 |                                                                                           |
| Output: Chapter Mapping JSON                                              | Output: Lesson Plan + LO Handoff                                | Output: Assessment Plan + Annotation Layer                                                |
+---------------------------------------------------------------------------+-----------------------------------------------------------------+-------------------------------------------------------------------------------------------+

The handoff between constitutions is always structured data, never free text. The LO handoff carries: implied LO text, competency target (C-code + weight), and chapter section. The Assessment Constitution reads this handoff alongside the Chapter Mapping JSON and Curricular Goals document — no other document is an input to assessment generation.
